## 1.4 Tone of Voice | نبرة الصوت

### The "Saudi White Dialect" Framework | إطار اللهجة البيضاء

Ziyada System communicates in **اللهجة البيضاء** (the "White Dialect") — a polished, universally understood Saudi Arabic that avoids hyper-regional slang while remaining warm and natural. It is NOT Modern Standard Arabic (فصحى). It is NOT Najdi, Hijazi, or Southern dialect exclusively. It is the shared professional Saudi voice.

زيادة سيستم تتواصل باللهجة البيضاء — لهجة سعودية مفهومة للجميع، مصقولة ومحترفة، بعيدة عن العاميّة المحليّة المفرطة. مو فصحى رسمية، ومو لهجة منطقة وحدة. هي الصوت السعودي المهني المشترك.

### Voice Attributes | سمات الصوت

| Attribute | Description EN | الوصف |
|-----------|---------------|-------|
| **Professional** | We speak with authority. We know our craft. No filler, no fluff. | نتكلم بثقة. نعرف شغلنا. بدون حشو. |
| **Warm** | We are approachable. We talk *with* clients, not *at* them. Use "نحن" and "معك". | نحن قريبين. نتكلم *مع* العميل مو *على* العميل. |
| **Direct** | Say it in one sentence. If it needs two, the first one was wrong. | قولها بجملة. لو تحتاج اثنتين، الأولى كانت غلط. |
| **Results-Oriented** | Every claim backed by a number. "زدنا المبيعات 34%" not "حسّنا الأداء". | كل ادعاء يرجع لرقم. |
| **Confident, Not Arrogant** | We let results speak. No superlatives without proof. | نخلي النتائج تتكلم. بدون مبالغات بدون دليل. |

### Copywriting Rules | قواعد الكتابة

**DO | افعل:**
- Use "أنت" (you) and "نحن" (we) — create a conversation, not a lecture.
- Lead with the pain point, then the solution. Never start with features.
- Use numbers: "وفّرنا 120 ساعة عمل شهرياً" is stronger than "وفّرنا وقت كثير".
- Keep sentences under 15 words in Arabic, 20 in English.
- Use Saudi business terms naturally: "عميل محتمل" (lead), "تحويل" (conversion), "عائد استثمار" (ROI).

**DON'T | لا تفعل:**
- Don't use فصحى constructs like "إنّ" / "لذا" / "بيد أنّ" — they create distance.
- Don't use English words when a Saudi Arabic equivalent exists. Say "أتمتة" not "أوتوميشن".
- Don't use humor or memes. We are premium, not playful.
- Don't use passive voice. "نحلل بياناتك" not "يتم تحليل البيانات".
- Don't start sentences with "نحن" repeatedly — vary the structure.

### Headline Formula | صيغة العناوين

```
[Pain Point] + [Solution] + [Quantified Result]
```

**Example:**
- EN: "Stop losing leads. Automate your funnel. Convert 3x more."
- AR: "وقّف ضياع العملاء. أتمت مسار المبيعات. حوّل 3 أضعاف."

## 2.2 Clear Space | المساحة الآمنة

The minimum clear space around the logo equals **the width of one hexagon vertex node** (approximately 1/8 of the logo width) on all four sides.

المساحة الآمنة حول الشعار تساوي **عرض نقطة عقدة واحدة من السداسي** (تقريباً 1/8 من عرض الشعار) على الجوانب الأربعة.

```
          ┌───────────────────────┐
          │    ↕ 1x clear space   │
          │  ┌─────────────────┐  │
          │  │                 │  │
          │←→│     LOGO        │←→│  1x each side
          │  │                 │  │
          │  └─────────────────┘  │
          │    ↕ 1x clear space   │
          └───────────────────────┘
```

## 2.4 Logo Dos and Don'ts | الاستخدام الصحيح والخاطئ

### DO | افعل

- Place the full gradient logo on dark backgrounds (`#0f172a` to `#1e293b`).
- Use the simplified blue version at sizes below 48px.
- Maintain the original aspect ratio (1:1 for the mark).
- Use the monochrome white version on photographic or complex backgrounds.

- ضع الشعار المتدرج الكامل على الخلفيات الداكنة.
- استخدم النسخة الزرقاء المبسطة للأحجام أقل من ٤٨ بكسل.
- حافظ على نسبة العرض الأصلية (١:١ للرمز).
- استخدم النسخة أحادية اللون البيضاء على الخلفيات المصورة أو المعقدة.

### DON'T | لا تفعل

| Misuse | Description | الوصف |
|--------|-------------|-------|
| **Rotation** | Never rotate the logo. The hexagon must remain upright. | لا تدور الشعار أبداً. |
| **Color Alteration** | Never change the gradient colors. Use only approved monochrome variants. | لا تغير ألوان التدرج. |
| **Stretching** | Never distort the aspect ratio. | لا تشوّه نسبة الأبعاد. |
| **Low Contrast** | Never place the gradient logo on a similarly-colored blue/purple background. | لا تضع الشعار المتدرج على خلفية زرقاء/بنفسجية مشابهة. |
| **Effects** | Never add extra drop shadows, outer glows, or bevels beyond the built-in filter. | لا تضف ظلال أو توهجات إضافية. |
| **Cropping** | Never crop any part of the hexagon or its nodes. | لا تقص أي جزء من السداسي أو نقاطه. |
| **Background Box** | Never place the logo inside a colored box or circle that isn't part of the approved lockups. | لا تضع الشعار داخل مربع أو دائرة ملونة غير معتمدة. |
| **Text Overlap** | Never overlay text on the logo mark. | لا تضع نصاً فوق الشعار. |

## 3.4 Brand Gradients | تدرجات العلامة التجارية

These are the signature gradient combinations. Use exactly as specified — do not improvise gradients.

هذي التدرجات اللونية المميزة للعلامة. استخدمها بالضبط كما هي — لا تخترع تدرجات جديدة.

### Primary Brand Gradient | التدرج الأساسي
```css
background: linear-gradient(90deg, #8b5cf6 0%, #3b82f6 50%, #06b6d4 100%);
```
Use: Brand bar, dividers, accent strips.

### Hero Text Gradient | تدرج عنوان الصفحة الرئيسية
```css
background: linear-gradient(270deg, #1e3a8a 0%, #3b82f6 40%, #7c3aed 70%, #1e3a8a 100%);
background-size: 300% 100%;
animation: gradientShiftHero 5s ease infinite;
```
Use: H1 headings on hero sections only.

### Neon Beam Border Gradient | تدرج حدود الشعاع النيوني
```css
background: linear-gradient(120deg, #3b82f6, #8b5cf6, #06b6d4, #ec4899, #3b82f6);
background-size: 260% 100%;
animation: borderBeam 2.5s linear infinite;
```
Use: Glass panel hover borders, featured card borders.

### CTA Button Gradient | تدرج زر الدعوة
```css
background: linear-gradient(135deg, #1e3a8a, #2563eb, #7c3aed, #1e3a8a);
background-size: 300% auto;
animation: shine 6s linear infinite;
```
Use: Primary call-to-action buttons only.

### Service Card Border Gradient | تدرج حدود بطاقة الخدمات

Use: Hovered service/feature cards only.

### Icon Box Gradient | تدرج صندوق الأيقونة
```css

```
Use: Icon containers, small badge backgrounds.

## 4.1 Font Families | عائلات الخطوط

| Language | Font | Fallbacks | CSS Variable | Source |
|----------|------|-----------|-------------|--------|
| **Arabic** | **Noto Kufi Arabic** | Tajawal, sans-serif | `--font-body-ar` | Google Fonts |
| **English** | **Plus Jakarta Sans** | Noto Kufi Arabic, sans-serif | `--font-body-en` | Google Fonts |

### Why These Fonts | لماذا هذه الخطوط

- **Noto Kufi Arabic**: Geometric Kufic letterforms project authority and modernity. Wide weight range (400–900) provides full typographic hierarchy. Excellent screen rendering at all sizes.
- **Plus Jakarta Sans**: Geometric sans-serif that harmonizes with Noto Kufi Arabic's proportions. Professional yet approachable. Supports extended Latin for international content.

نوتو كوفي عربي: حروف كوفية هندسية تعكس السلطة والحداثة. مدى أوزان واسع يوفر تسلسل طباعي كامل.
بلس جاكرتا سانس: خط هندسي بدون حواف يتناغم مع نسب نوتو كوفي العربي. مهني وقريب.

### Loading | التحميل

```css
@import url('https://fonts.googleapis.com/css2?family=Noto+Kufi+Arabic:wght@400;500;600;700;800;900&family=Plus+Jakarta+Sans:wght@400;500;600;700;800&display=swap');
```

## 5.5 Animation Reference | مرجع الحركات

All animations in the Ziyada System follow these principles:
- **Entrance**: Always from bottom (`translateY > 0`) with fade-in. Never from sides.
- **Easing**: `cubic-bezier(0.16, 1, 0.3, 1)` for reveals, `cubic-bezier(0.34, 1.56, 0.64, 1)` for bouncy pops.
- **Duration**: 0.4s–0.75s for UI transitions. Never exceed 1s for user-triggered animations.
- **Infinite loops**: Reserved for gradients and border beams only. Never animate content infinitely.

كل الحركات في زيادة سيستم تتبع هذه المبادئ:
- **الدخول**: دائماً من الأسفل مع تلاشي. أبداً من الجوانب.
- **السلاسة**: منحنيات بيزير محددة للكشف والقفز.
- **المدة**: ٠.٤ إلى ٠.٧٥ ثانية. لا تتجاوز ١ ثانية للحركات التفاعلية.
- **التكرار اللانهائي**: محجوز للتدرجات وحدود الأشعة فقط.

| Animation | Keyframes | Duration | Easing | Usage |
|-----------|-----------|----------|--------|-------|
| `heroTitleReveal` | opacity 0 + translateY(32px) + skewY(2deg) + blur(6px) → visible | `0.75s` | `cubic-bezier(0.16, 1, 0.3, 1)` | Hero H1 entrance |
| `heroFadeUp` | opacity 0 + translateY(22px) → visible | `0.6s` | `ease` | Hero subtitle, staggered |
| `heroBadgePop` | scale(0.8) + translateY(10px) → visible | `0.5s` | `cubic-bezier(0.34, 1.56, 0.64, 1)` | Hero badge/tag pop-in |
| `revealUp` | translateY(40px) → 0 | `0.7s` | `cubic-bezier(0.16, 1, 0.3, 1)` | Section content reveal |
| `statPop` | scale(0.7) + translateY → 1 | `0.55s` | `cubic-bezier(0.34, 1.56, 0.64, 1)` | Statistics counter pop |
| `pageFadeIn` | opacity 0 → 1 | `0.5s` | `ease` | Page transitions |
| `slideUp` | translateY(100%) → 0 | `0.4s` | `ease` | Detail view panels |
| `shine` | bg-position 200% → -200% | `6s` | `linear infinite` | CTA button gradient sweep |
| `borderBeam` | bg-position 0% → 100% → 0% | `2.5s` | `linear infinite` | Neon border on glass panels |
| `gradientShiftHero` | bg-position 0% → 100% → 0% | `5s` | `ease infinite` | Hero gradient text |

## 8.3 Icon Usage Rules | قواعد استخدام الأيقونات

| Rule | القاعدة |
|------|---------|
| Always use stroke-only icons. Never fill them. | دائماً استخدم أيقونات بحدود فقط. لا تملأها. |
| Maintain `stroke-width: 2` at the default 24px size. Scale proportionally. | حافظ على سمك الحدود ٢ عند ٢٤ بكسل. |
| At 16px, reduce stroke-width to `1.5`. At 32px+, increase to `2.5`. | عند ١٦ بكسل خفّض لـ ١.٥. عند ٣٢+ ارفع لـ ٢.٥. |
| Icons in UI inherit `currentColor`. Only override for gradient boxes. | الأيقونات في الواجهة ترث اللون الحالي. |
| Never use emoji or Unicode symbols as substitutes for Zia-Icons. | لا تستخدم إيموجي أو رموز يونيكود كبديل. |
| In bilingual layouts, icon position stays constant — only text direction changes. | موقع الأيقونة ثابت — فقط اتجاه النص يتغير. |

## 10.2 Mode Rules | قواعد الأوضاع

| Rule | القاعدة |
|------|---------|
| **Dark mode is the default** and primary brand expression. All new designs start in dark mode. | الوضع الداكن هو الافتراضي والتعبير الأساسي للعلامة. |
| Light mode is a **courtesy accommodation**, not an equal alternative. | الوضع الفاتح هو تسهيل، مو بديل متساوي. |
| Marketing materials, social media, and presentations always use dark mode. | المواد التسويقية والسوشال والعروض دائماً بالوضع الداكن. |
| Print may use light mode when dark printing is cost-prohibitive. | المطبوعات ممكن تستخدم الوضع الفاتح عند ارتفاع تكلفة الطباعة الداكنة. |
| Both modes must pass WCAG AA contrast requirements (4.5:1 for body, 3:1 for large text). | كلا الوضعين لازم يحققون معايير تباين WCAG AA. |
