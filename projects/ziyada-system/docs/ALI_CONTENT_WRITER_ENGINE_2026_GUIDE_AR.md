# دليل التشغيل الكامل: Ali Content Writer Engine 2026

## ملخص سريع (قبل البدء)
هذا النظام هو كاتب محتوى شامل (Universal Content Writer) يعمل على n8n ويستقبل بيانات أي نشاط تجاري، ثم:
1. يفهم النشاط من الإدخال + تحليل Apify.
2. يكتب محتوى مخصص للنشاط نفسه فقط.
3. يحافظ على أسلوب عربي سعودي احترافي بنبرة تشغيلية واضحة (مستوحاة من طابع Ziyada في الأسلوب فقط، بدون ربط المحتوى ببراند Ziyada).
4. يرسل التحديثات والملخصات عبر تيليجرام.
5. يسجل الحالة والمخرجات في Google Sheets ويدعم الروابط إلى Doc وSlides.

## الفكرة التشغيلية (Rigor + Executions)
الصرامة التشغيلية في هذا workflow تعتمد على 3 نقاط:
1. **مدخلات إلزامية قبل التوليد**: اسم النشاط، القطاع، الجمهور المستهدف، رابط النشاط، الموضوع.
2. **بوابة تشغيل واضحة**: التشغيل اليدوي من الشيت لا يتم إلا عند تعبئة خانة التريغر بقيمة صحيحة.
3. **تتبع تنفيذي**: كل تشغيل يسجل حالة التنفيذ، ملخص، وروابط المخرجات، مع سجل محادثة في Chat State.

## الأدوات المستخدمة داخل النظام
1. **n8n**: تنسيق كامل مسار الأتمتة.
2. **Telegram Trigger + Telegram Send**: استقبال الرسائل وإرسال التحديثات.
3. **Google Sheets**:
   1. Content Intake
   2. resault
   3. Chat State
4. **Apify**: تحليل السوق، اتجاهات المحتوى، وإشارات المنافسين.
5. **OpenAI**:
   1. طبقة محادثة ذكية (Telegram Conversation Manager)
   2. طبقة توليد المحتوى النهائي

## الخانة اليدوية الإلزامية لتشغيل التريغر من الشيت
التعبئة اليدوية الإلزامية تكون في:
1. **تبويب**: Content Intake
2. **العمود**: G
3. **النطاق**: G2:G
4. **اسم الحقل**: trigger_status

القيم المسموحة للتشغيل:
1. start
2. continue

> تم تمييز العمود G بالأصفر ووضع Validation بحيث يقبل start أو continue فقط.

## طريقة الاستخدام (تيليجرام)
### 1) بداية الحوار
يمكنك كتابة رسالة طبيعية مثل:
1. hi
2. السلام عليكم
3. ابدأ

### 2) إرسال بيانات النشاط
أفضل صيغة (واضحة وسريعة):
1. اسم الشركة: ...
2. القطاع: ...
3. الجمهور المستهدف: ...
4. رابط الموقع: ...
5. الموضوع: ...

يمكن أيضًا كتابة رسالة حرة طويلة، والنظام يحاول استخراج البيانات تلقائيًا.

### 3) بدء التنفيذ بعد اكتمال البيانات
اكتب:
1. ابدأ التنفيذ
أو
2. generate

### 4) طلب حالة أو ملخص
أمثلة أوامر طبيعية:
1. وين وصلنا
2. حالة
3. ملخص
4. روابط
5. status

## طريقة الاستخدام (من الشيت)
1. افتح Content Intake.
2. أضف صف جديد بالبيانات الأساسية.
3. في عمود trigger_status (G) اكتب start أو continue.
4. انتظر التنفيذ، ثم راقب النتائج في تبويب resault.

## ماذا يكتب النظام بالضبط؟
النظام يكتب محتوى **مرتبط بالنشاط المدخل فقط**:
1. لا يروّج لأي براند غير النشاط الموجود في الإدخال.
2. يعتمد على إشارات Apify لفهم السوق والأسلوب المناسب.
3. يلتزم بالعربية السعودية المهنية في الأسلوب.

## ملاحظة مهمة جدًا: ما الذي يجب تجنبه حتى لا يتوقف المسار؟
1. لا تغيّر أسماء الأعمدة الأساسية في Content Intake وresault وChat State.
2. لا تحذف تبويب Chat State.
3. لا تضع قيمة خاطئة في trigger_status (مثل done أو نص عشوائي) إذا هدفك التشغيل.
4. لا تمسح الروابط أو بيانات الاعتماد من إعدادات البيئة.
5. لا ترسل رسائل متكررة جدًا في ثوانٍ قليلة (قد تسبب تأخير مؤقت بسبب rate limit).
6. لا تعتمد على موضوع عام بدون بيانات نشاط؛ لازم تعريف النشاط أولًا.

## دورة التنفيذ الفعلية
1. استقبال الإدخال (Sheet أو Telegram).
2. بناء سياق النشاط.
3. (عند الحاجة) استكمال النواقص عبر المحادثة.
4. تحليل Apify.
5. توليد المحتوى (OpenAI).
6. كتابة النتائج في resault.
7. إرسال ملخص/حالة في Telegram.
8. تسجيل سجل المحادثة في Chat State.

## من المسؤول عن ماذا؟
1. المستخدم: إدخال بيانات نشاط دقيقة + تأكيد البدء.
2. النظام: جمع السياق، التوليد، التتبع، والتحديثات.

## أفضل ممارسة للتشغيل اليومي
1. أدخل بيانات نشاط واضحة من البداية.
2. تأكد أن الرابط يعكس هوية النشاط (موقع رسمي أو حساب رسمي).
3. بعد تلقي رسالة "البيانات واضحة" أرسل "ابدأ التنفيذ".
4. تابع الحالة عبر "وين وصلنا" أو "ملخص".

## تنبيه خاص بالنطاق
هذا المحرك **Universal Content Writer**، يعني يصلح لأي نشاط تجاري.
المرجع هنا فقط هو **نبرة الكتابة** (أسلوب عربي سعودي احترافي)، وليس ربط المحتوى بمشروع Ziyada أو أي مشروع آخر.
